# Project Instructions for AI Assistants

## Repository profile
- Monorepo with Angular 21 + NestJS 11.
- Use `pnpm` commands only.
- Follow issue-driven delivery with one active owner role (`agent:*`).
- Always start with a PM parent issue (`agent:pm`) that decomposes work into ordered child issues.
- For non-PM issues, require issue body fields:
  - `Parent PM: #<issue_number>`
  - `Execution Order: <positive_integer>`
  - `Depends on: #<issue_number>, ...` or `none`
- Governance rules source of truth: `governance/policy.json`.
- Run preflight before coding:
  - `pnpm agent:preflight -- --issue <issue_number> --agent <agent:role>`
- Branch naming must include issue id:
  - `feature/<issue_number>-<slug>`, `fix/<issue_number>-<slug>`, `chore/<issue_number>-<slug>`

## Angular development contract
- Use feature-first structure in `apps/web/src/app/features`.
- Respect layer boundaries: `ui -> state -> data-access`.
- Keep domain models and transport DTOs separated.
- Standardize state with `@ngrx/signals`.
- Keep UI components decoupled from stores and API services.
- Enforce accessible loading/error/empty/success states.

## Required validation
- `pnpm lint`
- `pnpm test`
- `pnpm build`
- Complete final self-review gate:
  - `docs/ai/checklists/ai-self-review-gate.md`
  - PR section `AI Self-Review Gate` with framework + decision
- Enforce framework selection based on changed scope:
  - `apps/web` => mark `Angular way`
  - `apps/api` => mark `NestJS way`

## Orchestration order
1. PM plans and sequences.
2. Assigned agents execute in order with one issue/branch/PR each.
3. QA/review gate validates quality before merge of each PR.

## Mandatory references
- Consult relevant docs/runbooks before implementation or review.
- Consult MCP and repository skills when they apply to the task.
